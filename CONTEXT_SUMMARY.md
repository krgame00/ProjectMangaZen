# ข้อมูลบริบทสำหรับโปรเจกต์ MangaZen (Context Summary)

กรุณาอ่านข้อมูลนี้อย่างละเอียดก่อนเริ่มช่วยเหลือฉัน นี่คือโครงสร้างและสถานะปัจจุบันของโปรเจกต์ "MangaZen"

## 1. เทคโนโลยีและโครงสร้างหลัก (Tech Stack & Core Architecture)
- **Framework:** Next.js 14 (App Router) ใช้งานร่วมกับ TypeScript
- **Styling:** Tailwind CSS + Vanilla CSS (กำหนดในไฟล์ globals.css)
- **Database & Auth:** MongoDB (Mongoose) + NextAuth.js
- **Hosting:** Vercel (ย้ายมาจาก Netlify เนื่องจากโควต้า Bandwidth เต็ม)

## 2. ระบบจัดเก็บไฟล์ (Storage & File Handling) - เน้นประหยัดต้นทุน
- **อัปโหลดรูปภาพ:** รูปภาพที่อัปโหลดจะถูกเก็บไว้ที่ **Telegram Bot API** เพื่อใช้งานเป็น CDN ฟรี
  - ควบคุมการอัปโหลดผ่านฝั่ง Client แบบเรียงลำดับทีละไฟล์ (`UploadModal.tsx`, `AddChapterModal.tsx`) เพื่อป้องกันการโดนแบนจากข้อจำกัด `429 Too Many Requests` ของ Telegram ส่วนการเรียงลำดับไฟล์ใช้การเรียงแบบตัวเลขตามธรรมชาติ (`localeCompare({ numeric: true })`)
- **ไฟล์ ZIP และ PDF:** ถูกเก็บไว้บน Google Drive
  - ระบบจะทำการดาวน์โหลดและแตกไฟล์ *แบบเรียลไทม์ (on the fly)* บนเบราว์เซอร์ของผู้ใช้โดยตรง ผ่านไลบรารี `jszip` (สำหรับ ZIP) และ `react-pdf` (สำหรับ PDF) เพื่อช่วยประหยัด Bandwidth ของเซิร์ฟเวอร์หลัก

## 3. การออกแบบ UI/UX (ประสบการณ์แบบแอปพลิเคชันพรีเมียม)
- **Design System:** ใช้ฟอนต์ `Inter` (สำหรับภาษาอังกฤษและตัวเลข) คู่กับ `Noto Sans Thai` ตกแต่งสไตล์แบบ Glassmorphism อย่างหนาแน่น (`backdrop-filter: blur`), รองรับ Dark Mode และปุ่มต่างๆ ใช้สีแบบไล่ระดับ (Vibrant Gradients)
- **Mobile-First Navigation:** ใช้แถบเมนูด้านล่าง (Bottom Navigation Bar) บนมือถือ คล้ายแอปพลิเคชัน Native พร้อมระบบค้นหาแบบซ่อนตัวที่สามารถกดขยายได้
- **Micro-Interactions:** ปุ่มกดและการ์ดมังงะทุกจุดมีเอฟเฟกต์การยุบตัวเมื่อถูกแตะ (`scale(0.96)`) และมีการขยับแบบสปริงตัว (`cubic-bezier(0.34, 1.56, 0.64, 1)`)
- **Smooth Loading:** รูปภาพทุกรูปมีโครงร่างจำลอง (Skeleton Loader แบบชีพจร) ระหว่างรอโหลด และจะค่อยๆ เฟดแสดงผลอย่างนุ่มนวลเมื่อโหลดเสร็จ เพื่อป้องกันเลย์เอาต์กระตุก

## 4. ระบบอ่านมังงะและการแปลด้วย AI (`ReaderUI.tsx`)
- **โหมดการอ่าน:** รองรับทั้งแบบอ่านทีละหน้า (จากซ้ายไปขวา / ขวาไปซ้าย) และแบบเลื่อนอ่านยาวแนวตั้ง (Webtoon-style)
- **Immersive Mode (โหมดเต็มจอ):** ผู้ใช้สามารถแตะที่จุดใดก็ได้บนหน้ามังงะเพื่อ ซ่อน/แสดง แถบเมนูด้านบนและล่าง
- **การแปลด้วย AI:** เชื่อมต่อกับ Gemini AI (2.5 Flash / 2.0 Flash) เพื่อแปลหน้ามังงะแบบเรียลไทม์
  - ผู้ใช้สามารถกดปุ่ม "✨ แปลหน้านี้" (Translate this page) ที่ลอยอยู่บนรูปภาพได้เลย
  - มีระบบ **"🎯 ลากแปล" (Manual Marquee Translation)** ให้ผู้ใช้วาดกรอบคลุมข้อความที่ AI แปลไม่ติดหรือแปลตกหล่นด้วยตัวเองบนหน้าจอ โดยระบบจะตัดภาพเฉพาะส่วนนั้นส่งให้ AI แปล และรวมผลลัพธ์ (Merge) เข้ากับข้อความเดิมบนหน้าจอ
  - มี **โหมด 18+ (Masking Mode)** ซึ่งระบบจะทำการหั่นรูปภาพออกเป็น 6 ส่วนก่อนส่งไปให้ AI แปล เพื่อหลบเลี่ยงระบบเซนเซอร์ตรวจจับเนื้อหาล่อแหลม และเพิ่มความแม่นยำในการอ่านข้อความเล็กๆ

## 5. สถานะปัจจุบัน
- หน้าตาของเว็บ (UI) เพิ่งได้รับการอัปเกรดครั้งใหญ่แบบ "Impeccable Overhaul"
- **การรองรับมือถือ (Mobile Responsive):** ปรับปรุงหน้าอ่านมังงะบนมือถือให้ใช้งานง่ายขึ้น (รองรับการทัชสกรีนวาดกรอบแปลข้อความแบบไม่เลื่อนหน้าจอ, ขยายความสูงของกล่องข้อความอัตโนมัติให้อ่านง่าย, และเอาแถบเมนูด้านล่างออกเพื่อไม่ให้บังเนื้อหา)
- โค้ดผ่านการ Type Check ถูกต้อง และ Deploy บน Vercel ผ่านเรียบร้อยแล้ว (อัปเดตล่าสุด: ฟีเจอร์ลากแปล และ Mobile UI)
